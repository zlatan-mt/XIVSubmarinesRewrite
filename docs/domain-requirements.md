# ドメイン要件ヒアリングメモ

## 目的
- 旧版利用者からの要望・痛点を整理し、リライト版の優先順位付けに活用する。
- M1 (Core+Infrastructure) 実装に必要な最低限の要件を明文化する。

## 既知の要望/課題
- **取得の安定性**: 工房入室時のメモリスキャンの信頼性向上、再試行制御。
- **外部連携**: Discord 通知の重複抑制、Notion 同期の競合対策。
- **UI/UX**: 情報密度の高い Debug タブの可読性向上、アラーム設定の分かりやすさ。
- **パフォーマンス**: スキャン/採用処理の CPU 使用率と GC 圧の軽減。
- **テレメトリ**: ログ収集と可視化の改善 (平均/分位数/エラー率の把握)。
- **設定管理**: 外部通知先などの秘匿情報を安全に保持する仕組み。

## M1 (Core + Infrastructure) に必要なタスク例
1. ドメインモデル定義 (Submarine, Voyage, Route, Alarm, Profile)。
2. データ取得インターフェース `IDataSource` とアルゴリズム設計 (Memory, UI fallback)。
3. 永続化コンポーネント (JSON bridge, Config) の抽象化。
4. ロギング/テレメトリ基盤 (構造化ログ、測定ポイント定義)。
5. 基礎テストプロジェクトの作成とユニットテスト構築。
